## 第一部分：编码规范检查

### 1.1 后端编码规范

- [ ] **代码格式**
  - [ ] 遵循Google Java Style Guide
  - [ ] 使用Lombok减少样板代码
  - [ ] 所有public方法有Javadoc注释
  - [ ] 常量定义在常量类中，不硬编码

- [ ] **命名规范**
  - [ ] 类名：UpperCamelCase（如 `UserService`）
  - [ ] 方法名：lowerCamelCase（如 `getUserById`）
  - [ ] 常量：UPPER_SNAKE_CASE（如 `MAX_RETRY_COUNT`）
  - [ ] 数据库表/字段：snake_case（如 `user_id`）
  - [ ] API路径：snake_case（如 `/api/novels/{id}`）

- [ ] **分层规范**
  - [ ] Controller层：只做参数校验和响应封装，不含业务逻辑
  - [ ] Service层：业务逻辑处理，事务管理
  - [ ] Repository层：数据访问，使用MyBatis-Plus
  - [ ] DTO层：数据传输对象，使用Lombok @Data

- [ ] **安全规范**
  - [ ] 密码使用BCrypt加密
  - [ ] SQL使用参数绑定，不用字符串拼接
  - [ ] 敏感信息不在日志中输出
  - [ ] 接口有权限注解 `@PreAuthorize`

### 1.2 前端编码规范

- [ ] **代码格式**
  - [ ] 遵循ESLint + Prettier配置
  - [ ] 使用有意义的变量/函数命名
  - [ ] 组件使用Composition API（`<script setup>`）
  - [ ] 避免使用any，使用明确的类型定义

- [ ] **项目结构**
  - [ ] API接口在 `api/` 目录集中管理
  - [ ] 组件在 `components/` 目录按功能分类
  - [ ] 工具函数在 `utils/` 目录
  - [ ] 类型定义在 `types/` 目录

- [ ] **性能规范**
  - [ ] 路由按需加载（懒加载）
  - [ ] 大列表使用虚拟滚动
  - [ ] 合理使用 `v-show` vs `v-if`
  - [ ] 避免不必要的响应式复制
