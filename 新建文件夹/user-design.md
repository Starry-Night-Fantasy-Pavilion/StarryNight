## 第一部分：用户端概述

### 1.1 设计定位

用户端（前台）是面向网文作者的核心创作平台，提供从创意到成品的完整创作体验。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 创作者优先 | 界面围绕创作流程优化，减少干扰 |
| 所见即所得 | 大纲/细纲/正文编辑直观可视化 |
| AI无缝衔接 | 随时唤醒AI，不打断创作心流 |
| 本地优先 | 数据本地存储，保护创作隐私 |

### 1.3 页面架构

```

├── 公开页面
│   ├── 首页 (落地页)
│   ├── 登录页

    ├── 作者中心
    │   ├── 创作概览

    ├── 作品管理
    │   ├── 作品列表
```

### 3.6 核心数据流

```
数据加载流程:

用户打开书籍
     │
     ▼
1. 检查本地缓存 (localStorage)
     │
     ├─ 有缓存 → 立即展示 (0延迟)
     └─ 无缓存 → 显示Loading
     │
     ▼
2. 发起API请求
     │
     ▼
3. API处理
     ├─ 获取书源配置
     ├─ 补全相对URL
     ├─ 注入认证信息(Cookie)
     ├─ 发起HTTP请求
     └─ 解析HTML/JSON响应
     │
     ▼
4. 核心解析
     ├─ cheerio解析HTML
     ├─ JSONPath提取JSON
     ├─ JavaScript动态处理
     └─ 清洗危险标签
     │
     ▼
5. 返回数据给前端
     │
     ▼
6. 存入本地缓存 + 展示UI
```
